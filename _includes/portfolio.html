WDI-16 Project Three
Plannerr App

Plannerr App

This was the first official group project during the GA course. We chose to build a mobile first "event" app that allowed users to sign up and login using their Facebook credentials.

Concept: Users can create an event and invite their friends, allowing each friend to suggest a different activity by utilising the Google Maps API.
Audience: Social savvy users who are looking for an app to arrange events and activities with their friends, without the clutter of irrelevant chat - Introducing Plannerr

The project is still being further developed with the addition of AngularJS and incorporating Ionic and Cordova to make it a hybrid mobile app.
Plannerr was created by: Finn Cavanagh, Gareth Gomersall, Adam Greenfield & Ben Alun-Jones

This was built using Node.js, Express and MongoDB. Utilizing facebook for social authentication, running the back-end API on Heroku and hosting the front-end on Github pages.

WDI-16 Project Two
Deskbrews

DeskBrews

One of my goals when choosing project 2 for WDI, was to learn how to use payment methods, one in particular Stripe, as I feel they are the fastest growing merchant provider and one of the easiest to implement (as I learnt during the development of this app). I wanted to create an app that will allow users to sign up and be able to order a product online within a few simple steps, and incorporate a seemless payment integration as well.

The Concept: An easy-to-use online beer ordering service.
The Audience: Companies/Organizations wanting to celebrate the end to a productive week with their staff or maybe even to help inspire new ideas - hence the name DeskBrews.

The app is responsive, thanks to the use of Bootstrap, and the payment process with Stripe is mobile formatted as well, so it's easy enough to use on a mobile device or tablet.

The app was built using Ruby on Rails, and utilizes the Stripe API for payment processing
